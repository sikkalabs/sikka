# Changelog

## 0.3.2

- Implemented Peer Health Scoring, Latency RTT Tracking (Exponential Moving Average), and Dynamic Ban Protection.
- Added penalty score tracking (`PenaltyTransientFailure`, `PenaltyInvalidPayload`, `PenaltyCryptoViolation`), `bannedUntil` timeouts, and latency-aware peer selection.
- Accelerated Proof-of-Work zero-bit counting using standard library hardware intrinsics (`math/bits.LeadingZeros8`).
- Optimized UTXO string key generation (`utxoKey`) to eliminate reflection and heap allocations in core DAG loops.
- Fixed test runner database lock deadlock in `internal/node/node_test.go` by adding explicit `Node.Close()` and test cleanup.

## 0.3.1

- Implemented O(1) Address Indexing for instant address UTXO and balance queries.

## 0.3.0
- Added `GET /metrics` Prometheus endpoint for Grafana node telemetry.
- Implemented O(1) LRU Caching and offloading for historical confirmed transactions to optimize memory scaling.
- Decoupled web wallet UI and WASM binaries in favor of official `sikkalabs/sikka-sdk`.
- Upgraded Go runtime toolchain to Go 1.23.0.
- Optimized Dockerfile with BuildKit cache mounts, ldflags binary stripping, ca-certificates, and HEALTHCHECK directive.
- Added multi-arch automated dev workflow (`dev.yml`).

## 0.2.0

- Added Deep Finality Guard (Witness Stripping) to permanently delete old ML-DSA-87 signatures, saving massive disk space.
- Added Spent UTXO Purge to completely drop fully-spent, confirmed UTXOs from memory and database.
- Added Weight Index Compaction to delete redundant saturated weight indices for deep transactions.
- Added `WitnessStripped` boolean flag to the Transaction struct for network sync compatibility.
- Changed `protocol_version` to `2` to enforce the new witness stripping rules.
- Changed `storageFormatVersion` to `"2"` to protect backwards-compatibility.
- Massively accelerated `GetBalance()` and `GetUTXOs()` queries by removing dead UTXOs from the search space.
